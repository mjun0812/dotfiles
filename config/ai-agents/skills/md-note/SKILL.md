# md-note

## Objective

このSkillは、**現在の会話セッションで整理・調査した内容**を、**日本語の Markdown 1 ファイル**にまとめ、**GitHub リポジトリへ保存**するためのものです。

新規調査は主目的ではありません。
まとめる際は会話内ですでに確立した情報を優先し、外部情報を勝手に補わないでください。

## Rules

ユーザーが明示的に上書きしない限り、必ず次を守ってください。

1. Markdownの先頭に目次を置くこと。
2. 再検索しなくてもMarkdownを読めば内容がわかる粒度で書くこと。
3. 追加検索や外部情報での補完は、ユーザーが明示的に依頼したときだけ行うこと。
4. まとめる際は会話内ですでに確立した情報を優先し、外部情報を勝手に補わないこと。
5. 参考リンクがあれば入れること。
6. 事実と推論と未確認事項を混同しないこと。
7. 1つのMarkdownファイルにまとめること。
8. Markdown作成後は確認なしで必ず GitHub へ保存すること。

## Steps

1. 対象範囲を確定する。指定がなければ会話全体。曖昧なら推定する。
2. 追加調査をせず、会話内の情報だけでテンプレートに沿ってMarkdownを書く。
3. Markdown本文をチャットに出す。
4. そのままGitHubへ保存する。Markdown作成後、確認なしで必ず保存する。
5. 成功時は保存先・commit message・URL(あれば)を報告。失敗時は理由を示す。
6. 次のセクションに従って最終チェックと返答を行う。

## 最終チェックと返答

最後の返答前に次を確認する:

- 目次が先頭にある
- Markdown単体で理解できる
- 結論が前半にある
- GitHub保存を実行した。

返答は「何をまとめたか / 保存結果」の2点に絞り、本文は再掲しない。
